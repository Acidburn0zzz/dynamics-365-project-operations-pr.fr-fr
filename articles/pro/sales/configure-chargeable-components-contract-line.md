<span data-ttu-id="333e0-105">Une ligne de contrat basée sur un projet a les composants *inclus* et *payant*.</span><span class="sxs-lookup"><span data-stu-id="333e0-105">A project-based contract line has *included* components and *chargeable* components.</span></span>

<span data-ttu-id="333e0-106">Les composants inclus sont des composants soumis à :</span><span class="sxs-lookup"><span data-stu-id="333e0-106">Included components are components that are subject to:</span></span>

  - <span data-ttu-id="333e0-108">Limites à ne pas dépasser</span><span class="sxs-lookup"><span data-stu-id="333e0-108">Not-to-exceed limits</span></span> 
  - <span data-ttu-id="333e0-109">Paramètres de fréquence de facturation définis sur la ligne de contrat basée sur le projet</span><span class="sxs-lookup"><span data-stu-id="333e0-109">Invoice frequency settings defined on the project-based contract line</span></span>

<span data-ttu-id="333e0-110">Un sous-ensemble des composants inclus peut être marqué comme payant à l'aide du champ **Type de facturation**.</span><span class="sxs-lookup"><span data-stu-id="333e0-110">A subset of the included components can be marked as chargeable using the **Billing Type** field.</span></span> <span data-ttu-id="333e0-111">Le champ **Type de facturation** est un ensemble d'options qui autorise les valeurs suivantes dans le contexte d'une ligne de contrat :</span><span class="sxs-lookup"><span data-stu-id="333e0-111">The **Billing Type** field is an option-set that allows the following values in the context of a contract line:</span></span>

  - <span data-ttu-id="333e0-112">Facturable</span><span class="sxs-lookup"><span data-stu-id="333e0-112">Chargeable</span></span>
  - <span data-ttu-id="333e0-113">Non facturable</span><span class="sxs-lookup"><span data-stu-id="333e0-113">Non-chargeable</span></span>

<span data-ttu-id="333e0-114">Les composants payants peuvent être définis sur les tâches, les rôles et les catégories de transaction.</span><span class="sxs-lookup"><span data-stu-id="333e0-114">Chargeable components can be defined on tasks, roles, and transaction categories.</span></span>
